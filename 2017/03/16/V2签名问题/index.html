signature scheme v2问题

signature scheme v2问题

发现

今天提测之后测试来了一个很神奇的问题，直接反馈是安装新包提示程序未安装，我经验中遇到不能安装的情况，无非就是系统版本不满足要求，检查代码后，排除了这个情况，但是这次我第一次遇到。

首先，从真机上并没有直观的反馈问题所在，于是我用adb命令安装了apk，得到了报错结果：INSTALL_PARSE_FAILED_NO_CERTIFICATES，这样就能知道问题和签名有关。
排除了缺少签名或者需要重新签名这种情况，那么问题在哪呢。

感谢google 让我找到了这个链接https://github.com/mcxiaoke/packer-ng-plugin/issues/31/看到了问题所在

了解

为了提高Android系统的安全性，Google从Android 7.0开始增加一种新的增强签名模式，从Android Gradle Plugin 2.2.0开始，构建系统在打包应用后签名时默认使用APK signature scheme v2，该模式在原有的签名模式上，增加校验APK的SHA256哈希值，如果签名后对APK作了任何修改，安装时会校验失败，提示没有签名无法安装，使用本工具修改的APK会无法安装，解决办法是在 signingConfigs 里增加 v2SigningEnabled false ，禁用新版签名模式，技术细节请看官方文档：https://developer.android.com/about/versions/nougat/android-7.0.html#apk_signature_v2/

解决

找到了问题来源那么让我们开始解决它吧，其实官网文档已经给出了解决方案，在build.gradle文件中signingConfigs里加入v2SigningEnabled false，问题就这么解决了么？我们打包试试，Build->Generate Signed APK，打完包重新安装，问题依然存在。

回过头看，我们在build.gradle中去配置，那么我们在界面化打包过程中并没有参与gradle的配置，所以我们需要用gradle命令打包

Gradle命令打包

参考：https://juejin.im/entry/577383c66be3ff006adc92a3/

打开AS底部的Terminal，敲行代码./gradlew assembleRelease，如果系统中没有安装Gradle，则会自动下载完成安装及初始化

在app–>build–>outputs–>apk路径中就可以看到打包成功后的APK

值得一提的是，不同渠道不同版本怎么打包，
举个例子来说明：

打包official渠道的release版本

gradlew assembleofficialRelease

打包official渠道的debug版本

gradlew assembleofficialDebug

打包official渠道的所有版本(Release+Debug+..)

gradlew assembleofficial

打全部渠道Release版本

gradlew assembleRelease
