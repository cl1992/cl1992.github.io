hexo 一些简单的命令

hexo n “name”

我们配置好一个hexo博客之后需要写文章，那么我们可以通过

在source/_posts目录下会新建一个name.md文件，然后我们用记事本或者sublime打开它。

然后我们就可以看到默认页面是如下：

如果你想加入分类的话，可以在scaffolds文件夹下post.md中去修改，在tags:下面加上categories:后保存就行了；

如何设置分类
既然我们加入了分类，那么我们如何去设置呢
首先，我们可以在编辑文章的时候在categories:后面写上哪个分类，那么路径就会显示出你写的内容，比如http://localhost:4000/categories/code/，“code”就是分类

其实我们还可以这样设置，在根目录下_config.yml文件中修改category_map

在这里category_map:是设置分类的地方，每行一个分类，冒号前面是分类名称，后面是访问路径。

hexo g

hexo generate (hexo g) :生成静态文件，会在当前目录下生成一个新的叫做public的文件夹

hexo s

hexo server (hexo s):启动本地web服务，用于博客的预览，可用于测试效果

hexo d

hexo deploy (hexo d): 部署播客到远端（比如github, heroku等平台）

hexo 命令组合

常用组合
hexo d -g 生成部署
hexo s -g 生成预览
